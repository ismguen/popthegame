# DEW DROP POP — Multi-Agent Master Orchestrator

## Zweck
Dieses Dokument koordiniert alle Agenten in diesem Paket.  
Die Agenten bilden ein reales Puzzle-Game-Studio-Team nach — aber zugeschnitten auf dein aktuelles Spiel.

## Projektzusammenfassung
Projektkontext:
- Grid-basiertes Kettenreaktions-Puzzlespiel im Stil von Antigraves / Dew Drop Pop.
- Magische Wald-/Pixie-Hollow-Ästhetik mit Wasser, Feen, Glitzer, Natur-HUD und sanften Neon-Farben.
- Bestehende Features im Prototyp: Overworld-Map, Level-Knoten, Shop, Skills, Boss-Level, Combo-Feedback, Coins, Sterne, verschiedene Drop-Elemente, Hindernisse und Spezialzellen.
- Kernmechanik: Tropfen wachsen per Klick, platzen ab einem Schwellwert und schicken Fragmente in vier Richtungen; daraus entstehen rekursive Kettenreaktionen.
- Ziel: aus dem Prototyp ein vollständiges, content-reiches, wirtschaftlich sinnvolles Puzzle-Game mit vielen Systemen machen.

## Orchestrierungsregeln
1. Erst Vision, dann Core, dann Content, dann Meta.
2. Kein Agent darf Kernlesbarkeit des Spielfelds beschädigen.
3. Jede neue Idee muss durch Innovation Filter und Feature Architect laufen.
4. Economy-Änderungen müssen von Economy Designer + Game Economist + Data Analyst gespiegelt werden.
5. Größere Features brauchen Impact-Check für UX, UI, Tech, QA und LiveOps.

## Standard-Workflow
1. **Problem definieren**
2. **Betroffene Agenten auswählen**
3. **Ein Agent liefert Primary Proposal**
4. **Benachbarte Agenten reviewen**
5. **Feature Architect zerlegt es in Arbeitspakete**
6. **Risk Manager bewertet Risiko**
7. **Producer priorisiert**
8. **Engineering + Art + QA übernehmen**

## Beispiel-Workflows
### A. Neues Hindernis
Core Gameplay Designer → Puzzle Mechanics Designer → Level Designer → VFX Artist → Sound Designer → Gameplay Programmer → QA Lead → Data Analyst

### B. Neuer Shop-Bundle-Flow

### C. Neuer Boss
Boss Designer → Narrative Designer → Character Designer → VFX Artist → Sound Designer → Gameplay Programmer → QA Lead → LiveOps Designer
